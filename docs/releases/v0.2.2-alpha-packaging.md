# v0.2.2-alpha-packaging

## Highlights

- Adds release packaging configuration for BookTrans Desk alpha builds.
- Adds GitHub Actions CI for build, tests, audit, and release safety checks.
- Adds alpha tester documentation, privacy notice, and issue template.
- Adds open source governance files.

## Features

- `npm run pack` creates an unpacked Electron app directory.
- `npm run dist` runs electron-builder distribution targets.
- `npm run release:check` validates build/test/audit and scans tracked files for release blockers.

## Security

- Release checks reject tracked `.env`, EPUB binaries, generated release/build folders, and common credential patterns.
- CI does not call real AI APIs, download EPUBs, run real EPUBCheck, upload user files, or build installers.

## Testing

- Existing unit and regression tests remain required.
- Fixture EPUBs are generated at test runtime and contain no commercial content.

## Known Limitations

- Windows packaging may require a native Windows environment for final installer validation.
- Alpha Windows builds are currently unsigned; expect operating-system trust prompts.
- macOS packaging is roadmap-only.
- The project remains an alpha focused on reflowable EPUB.

## Upgrade Notes

- Run `npm install` or `npm ci` to install electron-builder.
- Generated artifacts are written under `release/`, which is git-ignored.

## Alpha Tester Notes

Read `docs/alpha/ALPHA_TESTER_GUIDE.md` and `docs/alpha/PRIVACY_NOTICE.md` before testing with personal EPUB files.
